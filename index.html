Gemini-Powered Looker Agents

Built 6 AI agents on top of Looker Conversations using Gemini, giving non-technical stakeholders the ability to query DIP data in plain English. Each agent maps to a specific domain (sales, availability, search, etc.) with carefully constructed context so Gemini generates accurate Explore queries. This eliminates the ad-hoc request bottleneck — instead of waiting for an analyst to write LookML, partners and commercial teams get answers in seconds.

Experimentation Infrastructure

Built the end-to-end A/B testing infrastructure for a product search ranking model across 6 global regions. I designed the FACT and assignment SQL queries, configured experiment definitions and metrics in Eppo using unique user identifiers, and built the visualisation layer that showed whether new ranking variants outperformed the baseline. Results varied by region — some variants won, others didn't — and my reporting gave product teams the confidence to ship or kill each variant.

ABOUT

Data should be beautiful enough
to use voluntarily.

I believe the best analytics tools are the ones nobody calls "tools." They're just the place you go to understand what's happening in your business. If someone has to be trained to use a dashboard, the dashboard has failed. If a data point needs a footnote to explain itself, the data model is wrong.

This is the lens I bring to everything I build — from the data models underneath to the interfaces on top. Data infrastructure should be as carefully designed as the product that sits on it. I've learned that the gap between "technically correct" and "genuinely useful" is where most analytics work dies. I try to close that gap.

When I'm not building data products in Berlin, I'm probably reading AI research papers or optimising something that doesn't need optimising.
